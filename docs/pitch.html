Agora
The self-driving classroom

Autonomous AI agents that teach a goal end-to-end.

A multi-agent tutoring system for the Agentic AI in Education hackathon. Give it a learning goal; six agents plan, teach, diagnose, and adapt — with little to no human intervention.

The problem

The best teaching method doesn't scale.

One-to-one tutoring produces a two-standard-deviation improvement (Bloom, 1984). It is the most effective intervention we have in education — and the least scalable. There will never be enough expert tutors.

+2σone-to-one tutoringvsone teacher, thirty students
The insight

Tutoring is a loop, not a Q&A.

Most "AI tutors" just answer questions. A real tutor plans what to teach, notices confusion, diagnoses the underlying gap, and changes the plan to fix it.

plan→ teach→ assess→ diagnose→ adapt↺
The solution

Six autonomous agents run the whole loop.

One input — a learning goal. Everything else is decided by the system. In autopilot, a simulated learner even answers the quizzes, so a full session runs with zero human input.

The headline behavior

It rewrites its own plan.

When a wrong answer traces to a missing prerequisite, the Orchestrator injects a remedial micro-lesson it never originally planned, teaches it, then returns to the blocked concept. Nobody scripted it — it's a decision from the learner's state.

missed check→ misconception found→ plan rewritten→ gap closed

Engineering

Production-shaped, demo-safe.

Stack

  • Node + Express (tsx) · React control room
  • Redis: state, replayable event log, pub/sub, analytics
  • Real-time via SSE + Redis pub/sub — scales across containers
  • One-command deploy to Scalingo (+ Redis addon)

Why it never breaks on stage

  • Provider-agnostic LLM (Anthropic / OpenAI)…
  • …that degrades to a deterministic engine
  • No API key, no network needed for the demo
  • Typed end-to-end · unit + integration + smoke + browser tested
